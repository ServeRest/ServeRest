ServeRest

API Endpoint
https://serverest.dev

O ServeRest é um servidor REST que simula uma loja virtual com intuito de servir de material de estudos de testes de API.

Essa página documenta todas as rotas e como acessá-las. Para mais detalhes do ServeRest (como executar localmente, etc) acesse o GitHub do projeto.

Precisa de ajuda? Abra uma issue aqui.

Gostou do projeto? O que acha de apoiar? Veja como aqui.

Autenticação ¶

Para utilizar determinadas rotas é necessário estar autenticado.

Para autenticar pegue o valor de authorization da resposta de /login e o insira no header authorization da request que precisa de autenticação.

Rotas que requerem autenticação:

Verbo Rota Usuário tem que ser administrador?
POST /produtos Sim
PUT /produtos/{id} Sim
DELETE /produtos/{id} Sim
POST /carrinhos Não
DELETE /carrinhos/concluir-compra Não
DELETE /carrinhos/cancelar-compra Não

USUÁRIO ADMINISTRADOR

Algumas das rotas citadas acima requerem que o usuário seja administrador. Nessas rotas, além de estar logado, é preciso que o usuário do login possua a tag administrador igual a true.

Imagine esse comportamento igual a uma loja. Apenas funcionários podem alterar o cadastro dos produtos.

Timeout da autenticação ¶

A duração padrão do token retornado em authorization é de 600 segundos (10 minutos). Caso esteja expirado irá receber status code 401 (Unauthorized).

Aumente a duração do token para manter autenticado por um período de tempo maior sem realizar novo /login. Inicie o ServeRest da seguinte forma:

npx serverest --timeout <duracao-do-token-em-segundos>
npx serverest --timeout 3600

No exemplo acima estamos alterando o timeout para 3600 segundos (1 hora).

Bearer ¶

Por padrão o authorization retorna a palavra Bearer no início.

Caso queira desabilitar e receber apenas o token, inicie o ServeRest da seguinte forma:

npx serverest --nobearer
npx serverest -b

Atenção: Ainda será necessário enviar Bearer no header authorization ao utilizar rota que requer autenticação.

Cadastrar carrinho
POST/carrinhos

Requer autenticação. Saiba mais na seção Autenticação

O carrinho é vinculado ao usuário do token enviado no header Authorization.

É possível cadastrar apenas 1 carrinho por usuário.

Ao cadastrar carrinho com sucesso é feita a redução da quantidade no cadastro de cada produto inserido no carrinho.
